User Stories Taschenrechner

User Story Vorlage: Als <Rolle/ Anwendertyp> möchte ich <Ziel/ Wunsch/ Funktion>, um <Nutzen>

Titel: Beenden
Story: Als Benutzer möchte ich "Fertig" anstelle einer weiteren Zahl eingeben, um den Taschenrechner zu beenden
Akzeptanzkriterien:
	* Taschenrechner wird beendet, sobal "Fertig" (in genau dieser Schreibweise) eingegeben wird
	* es gibt einen entsprechenden Hinweis, dass der Benutzer weiß mit welchem Kommando er der Taschenrechner beenden kann
Status: 
	* offen

Titel: Fortlaufende Berechnung
Story: Als Benutzer möchte ich eine weitere Zahl eingeben, um die vorherige Operation nochmals auszuführen mit den vorherigen Resultat
Akzeptanzkriterien:
	* Gleitkommanzahlen zwischen -10.0 und 100.0 können eingegeben werden
	* die Operation aus der vorherigen Berechnung wird nochmals verwendet
	* das neue Resultat wird ausgegeben ohne weitere Interaktion mit dem Benutzer
Status:
	* offen

Titel: Objektorientierung
Story: Als Entwickler möchte ich die Anwendnung objektorientiert entwickeln, um später einfach weitere Funktionalitäten hinzufügen zu können
Akzeptanzkriterien:
	* Die Aufteilung der Funktionalitäten auf Klassen folgt dem Prinzip Separation of Concerns (SoC)
	* Bei der Erstellung von Klassen und Methoden wird das Prinzip Don't Repeat Yourself (DRY) eingehalten
Hinweis:
	* Diese Story ist eine Erweiterung zur Story "Softwarestruktur"
Status:
	* offen

Titel: Softwarestruktur
Story: als Entwickler möchte ich die Software sauber strukturieren, um später einfach weitere Funktionalitäten hinzufügen zu können
Akzeptanzkriterien: 
	* Das Single Level of Abstraction wird eingehalten oder eine Abweichung ist begründet und dokumentiert
Status:
	* umgesetzt
	* erledigt

Titel: Dividieren
Story: als Benutzer möchte ich eine Zahlen durch eine zweite dividieren
Akzeptanzkriterien:
	* Gleitkommazahlen zwischen -10 und 100 können multipliziert werden
	* Divsion durch 0 muss verhindert werden
Status:
	* umgesetzt
	* erledigt

Titel: Multiplizieren
Story: als Benutzer möchte ich 2 Zahlen miteinander multiplizieren
Akzeptanzkriterien:
	* Gleitkommazahlen zwischen -10 und 100 können multipliziert werden
Status:
	* umgesetzt
	* erledigt

Titel: Subtrahieren
Story: als Benuter möchte ich 2 Zahlen eingeben, um deren Differenz berechnen zu lassen
Akzeptanzkriterien: 
	* Gleitkommazahlen zwischen -10.0 und 100.0 können addiert werden
Status:
	* offen

Titel: Addieren
Story: als Benuter möchte ich 2 Zahlen eingeben, um deren Summe berechnen zu lassen
Akzeptanzkriterien: 
	* Gleitkommazahlen zwischen -10.0 und 100.0 können addiert werden
Status:
	* Implementiert
	* Getestet
	* Erledigt

Titel: Starten
Story: Als Benutzer möchte ich den Taschenrechner schnell aufrufen können, um mein Resultat schnell zu erhalten
Akzeptanzkriterien:
	* Die Anwendung wird innerhalb von 2 Sekunden auf einem Rechner gestartet
	* Die Anwendung läuft auf einem Rechner mit Windows 10
Status:
	* offen
